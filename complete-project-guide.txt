# 🚗 ParkGuard - Complete Full Stack Application Guide

## 📋 Project Overview

**ParkGuard** is a comprehensive vehicle parking management solution that addresses common parking issues through QR code technology, real-time notifications, and privacy-protected communication.

## 🚀 Complete Setup Guide

Ensure you have these installed:

3. **SMTP Email**
   - Gmail: Enable App Password
   - SendGrid: Get API key

## 🎯 Step-by-Step Installation

### Part 1: Backend Setup

```bash
# 1. Create project directory
mkdir parkguard && cd parkguard
mkdir backend && cd backend

# 2. Initialize Node.js project
npm init -y

# 3. Install dependencies (from provided package.json)
npm install express cors helmet express-rate-limit compression morgan \
  bcryptjs jsonwebtoken express-validator pg socket.io stripe twilio \
  nodemailer qrcode uuid swagger-jsdoc swagger-ui-express dotenv

# 4. Install dev dependencies
npm install --save-dev nodemon

# 5. Create directory structure
mkdir -p config middleware routes services database scripts

# 6. Copy all provided backend files to respective directories

# 7. Configure environment
cp .env.example .env
# Edit .env with your credentials

# 8. Setup PostgreSQL
createdb parkguard_db
psql parkguard_db < database/schema.sql

# 9. Start backend server
npm run dev
# Server runs on http://localhost:5000
# API docs at http://localhost:5000/api-docs
```

### Part 2: Frontend Setup

```bash
# 1. Navigate to project root
cd ..

# 2. Create Next.js app
npx create-next-app@latest frontend --typescript --tailwind --app

# 3. Navigate to frontend
cd frontend

# 4. Install dependencies (from provided package.json)
npm install axios socket.io-client @stripe/stripe-js @stripe/react-stripe-js \
  react-hook-form zod @hookform/resolvers react-hot-toast zustand date-fns \
  clsx tailwind-merge lucide-react qrcode html5-qrcode recharts \
  framer-motion js-cookie @tanstack/react-query

# 5. Copy configuration files
# - next.config.js
# - tailwind.config.js
# - tsconfig.json

# 6. Create directory structure
mkdir -p lib hooks components/{ui,layout,features,shared} types \
  app/{(auth)/login,(auth)/register,(dashboard)/dashboard,scan,shop}

# 7. Copy provided files
# - lib/api.ts
# - lib/store.ts
# - app/page.tsx

# 8. Configure environment
cp .env.local.example .env.local
# Edit with your API URL and Stripe key

# 9. Start frontend server
npm run dev
# App runs on http://localhost:3000
```

## 📱 Key Features Walkthrough

### Feature 1: Vehicle Registration with QR Code

**Backend Flow:**
1. User registers vehicle via `/api/vehicles` POST
2. Backend generates unique UUID
3. QR code image created with vehicle URL
4. Stored in database with vehicle record

**Frontend Flow:**
1. User fills vehicle form
2. API request sent
3. QR code displayed immediately
4. Download/print options available

### Feature 2: Incident Reporting

**Backend Flow:**
1. Public endpoint `/api/incidents/report` receives report
2. Validates vehicle exists
3. Creates incident record
4. Sends multi-channel notification to owner

**Frontend Flow:**
1. Scanner detects QR code
2. Fetches vehicle info
3. Shows action buttons
4. Submits selected incident type
5. Confirmation shown

### Feature 3: Privacy-Protected VoIP

**Backend Flow:**
1. Twilio creates conference room
2. Calls owner first with message
3. Waits 3 seconds, calls reporter
4. Both connect to conference
5. Numbers remain hidden

**Frontend Flow:**
1. User clicks "Contact Owner"
2. Enters their phone number
3. Receives call shortly
4. Connected to owner privately

### Feature 4: E-commerce Integration

**Backend Flow:**
1. Stripe checkout session created
2. User redirected to Stripe
3. Webhook confirms payment
4. Order status updated
5. Email confirmation sent

**Frontend Flow:**
1. Browse QR sticker products
2. Add to cart
3. Proceed to checkout
4. Stripe payment form
5. Order confirmation page

## 🔒 Security Best Practices

### Backend Security
✅ JWT tokens with expiration
✅ Bcrypt password hashing (12 rounds)
✅ Rate limiting on all endpoints
✅ SQL injection prevention (parameterized queries)
✅ Input validation with express-validator
✅ CORS configuration
✅ Helmet.js security headers
✅ Environment variable protection

### Frontend Security
✅ HttpOnly cookies for tokens
✅ XSS prevention (React escaping)
✅ CSRF protection
✅ Secure password validation
✅ API key hiding (env variables)
✅ Input sanitization
✅ Protected routes
✅ SSL/HTTPS in production

## 🚀 Deployment Guide

### Backend Deployment (Heroku)

```bash
# 1. Install Heroku CLI
# 2. Login
heroku login

# 3. Create app
cd backend
heroku create parkguard-api

# 4. Add PostgreSQL
heroku addons:create heroku-postgresql:hobby-dev

# 5. Set environment variables
heroku config:set JWT_SECRET=your_secret
heroku config:set STRIPE_SECRET_KEY=your_key
# ... set all environment variables

# 6. Deploy
git init
git add .
git commit -m "Initial commit"
git push heroku main

# 7. Run migrations
heroku run npm run migrate
```

### Frontend Deployment (Vercel)

```bash
# 1. Install Vercel CLI
npm i -g vercel

# 2. Login
vercel login

# 3. Deploy
cd frontend
vercel

# 4. Set environment variables in Vercel Dashboard
# NEXT_PUBLIC_API_URL=https://parkguard-api.herokuapp.com
# NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...

# 5. Deploy to production
vercel --prod
```

## 🆘 Troubleshooting

### Common Issues

**Database Connection Failed**
```bash
# Check PostgreSQL is running
sudo systemctl status postgresql
# Or on Mac
brew services list

# Test connection
psql -U parkguard_user -d parkguard_db
```

**Twilio Not Working**
- Verify credentials in .env
- Check phone number format (+1234567890)
- Ensure webhook URLs are public

**Stripe Webhooks Failing**
```bash
# Use Stripe CLI for local testing
stripe listen --forward-to localhost:5000/api/orders/webhook
```

**Socket.IO Not Connecting**
- Check CORS configuration
- Verify WebSocket support
- Check firewall settings

## 📚 API Documentation

Full interactive API documentation available at:
**http://localhost:5000/api-docs** (when backend is running)

### Quick Reference

**Authentication**
- `POST /api/auth/register` - Register user
- `POST /api/auth/login` - Login
- `GET /api/auth/profile` - Get profile

**Vehicles**
- `GET /api/vehicles` - List vehicles
- `POST /api/vehicles` - Add vehicle
- `PUT /api/vehicles/:id` - Update vehicle

**Incidents**
- `POST /api/incidents/report` - Report incident (public)
- `GET /api/incidents/my-reports` - Get my reports
- `PUT /api/incidents/:id/acknowledge` - Acknowledge

**Orders**
- `POST /api/orders/create-checkout` - Create checkout
- `GET /api/orders/my-orders` - Get orders
